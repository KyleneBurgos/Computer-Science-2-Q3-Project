# Computer-Science-2-Q3-Project
# PROJECT TITLE: The Equation Chronicles: A Math RPG Game
## PROJECT DESCRIPTION
### 	This project combines the intricate concepts of Mathematics with the entertainment from video games. Its main audience consists of Elementary to Middle school students that struggle with balancing gaming and studying. It revolves around the player exploring the map of the game while solving mathematical equations in a story-like manner. It aims to combine the opposite qualities of both genres to make learning more enjoyable without it feeling draining or exhausting. The different stage levels that align with the player’s grade level indicates that the project isn’t limited to basic arithmetic but also contains elementary algebra and middle school mathematics. The project expects that all players would be able to enjoy the beauty of mathematics with a method they’re already familiar with for easier balance with their academic life and internet access.

## PROJECT OBJECTIVES
### 1. To help students find enjoyment in studying difficult subjects, specifically Mathematics, by combining fun and exciting story games with math problems, where difficulty and topics depend on the student’s grade level. This project should act as a fun learning material for Math, mixing games with academics.
### 2. To balance students’ time for playing online games and studying Mathematics by combining both into one program. This program allows students to play and enjoy while also learning or studying Math. 
### 3. To build students’ interest in studying by mixing games, which the new generation enjoys, with Math problems. By this, students will be entertained, building their interest in both the story and answering the Math problems, increasing the chance of more study sessions.
### 4. To boost students’ academic performance by providing them with a program that makes learning Mathematics fun, interesting, and exciting.


## PLANNED FEATURES AND REQUIREMENTS:

### CORE FEATURES:
#### 1. Entry Point - This feature is the first thing the player will see after opening the game. It contains the log in, sign up, or play as a guest options that later on decides if their data gets saved or not. The player sets their username and creates their password when they pick sign up and saves the data for when they have to log back in.
#### 2. Splash Screen - After the player chooses their account of choice, they would be led to a screen that shows the options terms and conditions or mechanics of the game. The player has the option to check the mechanics of the game to understand how to play and how to properly navigate around the other features of the code.
#### 3. Main Menu - This serves as the overall beginning of the game as it contains the “Play” button and the settings for comfortable gameplay. 
#### 4. Levels - It is the feature that determines the difficulty level of the equations the player may face. It has arithmetic and other sub topics pf mathematics that vary from the level the player is it. It has elementary equations up to middle school equations for the target audience.
#### 5. Input Validation - This feature allows the player to know whether their answer is wrong or not. When the player inputs an answer for the equation given, this feature identifies if it’s correct and adds points or incorrect and makes the player lose a life in the game.
### ADDITIONAL FEATURES:
#### 1. Limited Hearts - This feature exists to make players be more challenged and use their full potential to not lose their lives and reset the level they’re playing at. It shows the amount of tries they have left for the game.

## HOW TO RUN THE PROGRAM
### 1. Download a Python Compiler to access the code normally. If a compiler is already downloaded then proceed to the next step.
### 2. Access the code and it may run successfully with no errors.
### 3. Users would be met with the entry point and either choose to log in, sign up, or play as a guest. For beginners, it’s recommended to sign up and save the password for easy log in the next time the code is accessed.
### 4. After signing up/logging in or choosing the guest option, the user is met with a splash screen that shows the mechanics of the game. It’s recommended to read the mechanics first if the user is a beginner of the game. 
### 5. After learning the mechanics, the user is brought to the main menu that contains the play button and the settings. To begin the game, choose the “Play” option.
### 6. Input the answer of the given equations and the Input Validation feature decides if it’s correct.
### 7. A correct answer gives the user points while an incorrect answer subtracts a life from their limited tries. 
### 8. Finish the level and check the score to see how you did on that level.
